History.

2017
PHP DevelStudio was the first "IDE" where I started to learn programming
That "IDE" used PHP as a programming language
The first time when I heard about PHP DevelStudio it was like the beginning of the 2017
Seeing how easy it is to create a program (although it still can be decompiled)
I was thinking to create a few programs
I forgot about PHP DevelStudio until November 2017, then I decompliled the first program
On 1st December 2017, I was thinking of creating a launcher, and I made a few updates and gave it up(I gave up, because I wanted to improve the user interface and the code)

The rising.

2018

In the beginning of 2018 like January, I created Calculator, Task Manager (that can only kill process though), Base64 Encoder/Decoder, and i've created Unlocker
In the summer of 2018, I created the first decompiler VDecompiler that could decompile basic sections (Events, Config, Forms, Loader(Not always))
In the autumn of 2018, I created another decompiler VDecompiler 2.0, but he could decompile Protectors, except for specific (E.g when someone creats a custom protection with custom sections)
In the middle of autumn 2018, I created VDecompiler 2.0 and I learned a lot about DevelStudio Protection (Loader: A seperate file/section of program where the events/configs/forms load, and knowing how and where they decode you can get the code, How to get code of program without a decompiler or injector: Hook does a lot, How the injector gets code: While injected program is running, it replaces php5ts.dll original to php5ts.dll injected, after this the injected dll begins to hook as much as possible code, but there's a possibility to protect the program e.g check the checksum of dll)
In October 2018, I created "obfuscator" program that was replacing strings to "IIIIIII", or "OOOOOOOO", "eval(base64_decode("
At the end of 2018, I created "useless" the program that was used to test something (I can't recall)
It was the end of 2018, I was thinking what to create, and it was TrashLauncher 2.0 (i still didn't know how to write a program completely and took a lot from the internet)

The start to falling.

2019

Can't say what happened at the beginning of 2019
In the summer of 2019, I've created TrashLauncher 2.0, but it had many bugs, and its protection wasn't good
In the end of 2019, and as always I've created TrashLauncher 2.0 (I don't know what for I created it then)

The Fall.

2020

Since 2020, I have figured out how to decompile any program that was made by PHP DevelStudio
The year when I decided to leave PHP DevelStudio compelety, because the community switched to DevelNext or switched to another programming language
And then I learned Java, so I could write programs in Java
In the end of 2020, I've created TrashLauncher 2.0 (UPD: There's a new launcher called TrashLauncher 2.0 written using C# + C++, the current one won't be rewritten). UPD2: TrashLauncher 2.0 is no longer supported

Postscript.

2021

By this time, DevelStudio is useless program without future
A lot had changed before that
The same relates to DevelNext

Status of projects: Calculator: No longer supported: 2018.02 (2018/2), Task Manager: No longer supported: 2018.02 (2018/2), Base64 Encoder/Decoder: No longer supported: 2018.02 (2018/2), Unlocker: No longer supported: 2018.01 (2018/1) VDecompiler: No longer supported: 2018.10 (2018/10), VDecompiler 2.0: No longer supported: 2018.10 (2018/10), TrashLauncher 2017: No longer supported: 03.12.2017, Useless: No longer supported: 2018.12 (2018/12) TrashLauncher 2018: No longer supported: 31.12.2018 , TrashLauncher 2019: No longer supported: 2019.12 (2019/12)

History.

2018
I tried DevelNext in January 2018, as usual I didn't know how to write more complex program than hello world (lol)
For sure, DevelNext is definitely better than PHP DevelStudio at compiling the classes to bytecode thereby making it almost impossible to decompile the code (There was one decompiler, but he doesn't work since DevelNext added protection)
In spring 2018, I've created Calculator 2.0 with a new interface and he supported cross-platform running (you could run him on Windows, Linux, MacOS)
In summer 2018 or so, I've learned a lot about DevelNext (History: When was DevelNext created, Syntax: After all, I didn't like the syntax, Protection: You need to be a good re-engineer to dissassemble any program written in DevelNext, Compiling: Nothing special, the compiler during the compilation builds every file to .jar, then it "obfuscates" the program, Ui: Not gonna lie, You can create a nice UI with a modern look, but then we would be faced with high memory usage, it's still Java with a different wrapper)
The following is not without TrashLauncher that was created on DevelNext with a new dark interface and good protection from decompilers
TrashLauncher existed until the end of 2018

Status of projects: Calculator 2.0: No longer supported: 2018.03 (2018/3), TrashLauncher: No longer supported: 2018.12 (2018/12)
Note: The history is over.

It could have been a good IDE for creating cross-platform programming, but I didn't like it for some reason

History.

2021

On January, 2021, I was thinking of redesigning the launcher, because he looked like a program from 2000-2009 and the launcher was complelety written on Java (As a result of this: Terrible performance, memory consumption, non-native look, laggy JVM, need to have Java, Garbage collection that can't be used manually, etc...)

With this language I have created TrashLauncher 2.0 with a dark theme and user-friendly GUI, it became a good idea tbh
